# 드롭 타겟 × 파일 셸프 통합 — 사용자 시나리오 & UX 설계 (PoC)

브랜치: `poc/shelf-drop-integration`
상태: 설계/목업 (구현 전)

## 1. 배경 — 현재 두 개의 플로팅 패널

| 패널 | 형태 | 역할 | 성격 |
|------|------|------|------|
| **드롭 타겟**(DropTargetView) | 자유 부유형 **콤팩트 필**(224×60, 어디든 이동) | Finder에서 파일을 놓으면 드롭 폴더로 복사 → **자동 변환** | **입력구**(convert-in) |
| **파일 셸프**(ShelfView) | **왼쪽 엣지-도킹**(핸들 22 + 트레이 450, 높이 220) | 파일을 임시로 담아 두고 원하는 위치로 **드래그 아웃**(이동/복사) | **임시 보관대**(hold & move-out) |

문제의식: 화면에 **떠 있는 창이 둘**이라 시선·공간이 분산되고, "변환은 필, 이동은 셸프"라는 두 진입점을 사용자가 각각 기억해야 한다. 두 기능은 모두 "파일을 잠깐 이 앱에 얹는다"는 공통 제스처를 쓰므로, **하나의 엣지-도킹 표면**으로 합칠 여지가 크다.

## 2. 통합 컨셉 — 하나의 엣지 패널, 두 개의 존(zone)

왼쪽 가장자리에 도킹된 **단일 패널**. 접히면 얇은 탭, 펼치면 위아래 두 존:

1. **변환 드롭존**(상단) — 구 드롭 타겟. 영상·이미지를 놓으면 변환.
2. **보관 트레이**(하단) — 구 파일 셸프. 아무 파일이나 담아 두고 드래그 아웃.

핵심 원칙:
- **P1. 한 표면, 명확한 두 존** — 통합하되 "변환/보관"의 의미가 시각적으로 분리돼야 한다(색·아이콘·라벨·구분선).
- **P2. 드래그 중 목표 존 강조** — 파일을 끌고 오면 커서가 놓인 존만 하이라이트되어 "여기 놓으면 무엇이 되는지" 즉시 보인다.
- **P3. 기존 근육기억 보존** — 왼쪽 가장자리로 끌면 펼쳐지고, 트레이에서 끌어내면 이동한다(셸프 동작 유지).
- **P4. 통합의 부가가치** — 변환 결과를 곧바로 보관 트레이에 얹어, "여기서 변환 → 여기서 결과를 원하는 곳으로" 한 표면에서 완결(선택 기능, 아래 S5).

트레이드오프: 드롭 타겟의 **자유 이동(어디든 배치)** 자유도는 포기하고, **엣지 고정 단일 표면**의 정돈됨을 택한다. (드롭 타겟을 자주 화면 중앙에 두던 사용자에겐 손실 → 설정으로 구 방식 유지 옵션 고려.)

## 3. 사용자 시나리오

### S1. 빠른 변환 (구 드롭 타겟 역할)
1. 사용자가 Finder에서 화면 녹화 `.mov`를 왼쪽 가장자리 탭으로 끌고 간다.
2. 탭에 다가가자 패널이 **펼쳐지고 상단 "변환 드롭존"이 파랑으로 강조**된다.
3. 드롭존에 놓으면 "변환 시작" 플래시 → 변환이 돌고, 완료 시 (설정에 따라) 출력 폴더가 열린다.
- **성공 기준**: 별도 창을 찾지 않고 왼쪽 가장자리 한 곳으로 끌어 변환이 시작된다.

### S2. 임시 보관 후 이동 (구 셸프 역할)
1. 여러 폴더에 흩어진 파일을 하단 **보관 트레이**로 하나씩 끌어 담는다(원본 참조, 복사본 안 쌓임).
2. Finder에서 목적 폴더를 연 뒤, 트레이에서 **여러 개를 선택해 드래그 아웃** → 이동(같은 볼륨)·복사(다른 볼륨/⌥).
3. 이동으로 빠져나간 카드는 트레이에서 자동으로 사라진다.
- **성공 기준**: 변환존과 헷갈리지 않고 보관·이동을 수행한다.

### S3. 두 존 구분 (핵심 UX 리스크 해소)
1. 파일을 끌고 패널 위로 진입한다.
2. **커서가 상단이면 변환존만**, **하단이면 트레이만** 테두리·배경이 강조되고 라벨이 "여기 놓으면 변환/보관"으로 바뀐다.
- **성공 기준**: 놓기 전에 결과(변환 vs 보관)를 확신할 수 있다. 잘못 놓을 확률 최소화.

### S4. 접힘 상태에서의 드롭
1. 패널이 접혀 얇은 탭만 보인다.
2. 파일을 탭으로 끌면 **자동으로 펼쳐지고**, 두 존이 드러난 상태에서 원하는 존에 놓는다.
- 탭 자체는 특정 동작을 **확정하지 않는다**(모호함 방지). 펼침→존 선택이 정식 경로.
- **성공 기준**: 접혀 있어도 두 기능 모두 접근 가능하고, 오작동(원치 않는 변환)이 없다.

### S5. 통합 부가가치 — 변환 결과를 바로 옮기기 【확장】
1. 변환존에 파일을 놓아 변환한다.
2. 완료되면 결과 파일 카드가 **보관 트레이 상단에 자동으로 얹힌다**(잠깐 하이라이트).
3. 사용자는 그 결과 카드를 곧바로 원하는 폴더로 드래그 아웃한다.
- **성공 기준**: "변환 → 결과 배치"가 창 전환 없이 한 표면에서 끝난다. (PoC에서는 토글로 on/off)

### S6. 다중 모니터·화면 밖 방지 (기존 보장 유지)
- 접힌 탭은 **마우스가 있는 화면**의 왼쪽 가장자리를 따라간다.
- 모니터 구성이 바뀌어도 패널이 화면 밖으로 나가지 않게 위치를 보정한다.

## 4. 우려 시나리오 & 대응

| # | 우려 | 대응 |
|---|------|------|
| C1 | 변환존과 트레이가 세로로 붙어 어디 놓을지 혼란 | P2 존 하이라이트 + 넉넉한 구분선/여백, 드래그 중 라벨 전환 |
| C2 | 변환존에 변환 불가 형식(zip 등)을 놓음 | 존이 거부(빨간 힌트) 또는 "보관으로 담을까요?" 폴백 — PoC 기본: **거부 + 힌트** |
| C3 | 트레이에 영상/이미지를 "변환하려" 잘못 놓음 | 트레이는 항상 보관만. 변환은 상단존 전용(라벨로 명확화) |
| C4 | 패널이 커져 화면 세로를 많이 먹음 | 접힘 기본 유지, 펼침 높이 ~320로 제한, 트레이는 스크롤 |
| C5 | 기존 드롭 타겟 자유 배치를 쓰던 사용자 이탈 | 설정에 "드롭 타겟 분리 모드"(구 방식) 유지 옵션 |
| C6 | 접힌 탭 드롭의 모호성 | S4: 탭은 펼침 트리거만, 동작 확정 안 함 |

## 5. 레이아웃 스펙(초안)

- 엣지-도킹 좌측, 우측 모서리만 라운드(24). 글래스(VisualEffect)+검정 24% 오버레이. 그라디언트 `#0EA5E9→#6366F1→#8B5CF6`.
- **접힘**: 핸들 22폭. 상단 스택 아이콘 + 하단 카운트 배지. (드래그 호버 시 펼침)
- **펼침**: 폭 ≈ 22(핸들) + 460(내용). 높이 ≈ 320.
  - 상단 **변환 드롭존**: 높이 ~104. 큰 아이콘 배지 + "드롭하여 변환 / 영상·이미지". 드래그 타겟 시 확대·글로우.
  - **구분선** + 섹션 라벨.
  - 하단 **보관 트레이**: "파일 보관" 헤더(카운트/전체 지우기) + 카드 그리드(스크롤). 비었을 때 안내 문구.

## 6. 구현 방향(요약, 다음 단계)

- `ShelfController`/`ShelfView`에 **변환 드롭존 서브뷰**를 상단에 추가하고, 드롭존은 `WatchCoordinator.ingest(urls:)`를 호출(기존 로직 재사용).
- 존별 `onDrop`/hover 상태 분리로 P2/P3 구현. `ShelfCollectionView`는 그대로 하단에 배치.
- `DropTargetController`는 유지하되 통합 모드에서는 숨김(설정으로 분리 모드 선택 가능, C5).
- S5(결과 자동 얹기)는 `finish(outcome:)`에서 성공 시 결과 URL을 `ShelfStore`에 삽입(토글).
- 브랜치 PoC이므로 우선 **통합 화면 렌더링 + 두 존 드롭 동작**까지를 1차 목표로 한다.
