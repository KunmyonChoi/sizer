03

사용자 시나리오

각 흐름과 "성공 기준".

S1빠른 변환

Finder에서 녹화 파일을 왼쪽 가장자리 탭으로 끌면 패널이 펼쳐지고 상단 변환존이 강조 → 놓으면 변환 시작, 완료 후 출력 폴더 열림.

별도 창을 찾지 않고 한 곳으로 끌어 변환이 시작된다.

S2임시 보관 후 이동

흩어진 파일을 하단 보관 트레이에 모아 담고, 목적 폴더로 여러 개를 선택해 드래그 아웃(이동/복사). 빠져나간 카드는 자동으로 사라진다.

변환존과 헷갈리지 않고 보관·이동을 수행한다.

S3두 존 구분

파일을 끌고 진입하면 커서가 상단이면 변환존만, 하단이면 트레이만 강조되고 라벨이 "여기 놓으면 변환/보관"으로 바뀐다.

놓기 전에 결과(변환 vs 보관)를 확신할 수 있다.

S4접힘 상태 드롭

얇은 탭에 파일을 끌면 자동으로 펼쳐지고, 두 존이 드러난 상태에서 원하는 존에 놓는다. 탭 자체는 동작을 확정하지 않는다.

접혀 있어도 두 기능 모두 접근, 오작동 없음.

S5변환 결과 바로 옮기기 · 확장

변환이 끝나면 결과 카드가 보관 트레이 상단에 자동으로 얹히고(잠깐 하이라이트), 사용자는 그 카드를 곧바로 원하는 폴더로 끌어낸다.

"변환 → 결과 배치"가 창 전환 없이 한 표면에서 끝난다.

S6다중 모니터·화면 밖 방지

접힌 탭은 마우스가 있는 화면의 왼쪽 가장자리를 따라가고, 모니터 구성이 바뀌어도 패널이 화면 밖으로 나가지 않게 위치를 보정한다.

어떤 화면 배치에서도 항상 닿는다(기존 보장 유지).
04

우려 & 대응

통합에서 생길 수 있는 혼란과 설계적 방어.

#우려대응
C1두 존이 붙어 어디 놓을지 혼란드래그 중 목표 존만 하이라이트 + 넉넉한 구분선, 라벨 전환
C2변환존에 변환 불가 형식(zip 등)존이 거부(빨간 힌트). 기본: 거부, 옵션으로 "보관으로 담기" 폴백
C3트레이에 영상을 "변환하려" 잘못 놓음트레이는 항상 보관 전용. 변환은 상단존만(라벨로 명확화)
C4패널이 커져 세로 공간 차지접힘 기본 유지, 펼침 높이 제한, 트레이 스크롤
C5드롭 타겟 자유 배치를 쓰던 사용자설정에 "드롭 타겟 분리 모드"(구 방식) 유지 옵션
C6접힌 탭 드롭의 모호성탭은 펼침 트리거만, 동작 확정 안 함(S4)
